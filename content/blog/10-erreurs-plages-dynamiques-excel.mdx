# 10 Erreurs avec les Plages Dynamiques

Evitez ces pieges courants avec les fonctions de tableaux dynamiques.

## Erreur 1 : Espace insuffisant

### Probleme

#PROPAGATION apparait car des cellules ne sont pas vides.

### Solution

- Liberez l'espace sous la formule
- Ou utilisez PRENDRE pour limiter les resultats

## Erreur 2 : Reference a la plage entiere

### Probleme

```excel
=FILTRE(A:D;B:B="Paris")
```

Peut etre lent ou retourner des erreurs.

### Solution

Delimitez la plage :

```excel
=FILTRE(A2:D1000;B2:B1000="Paris")
```

## Erreur 3 : Oublier si_vide dans FILTRE

### Probleme

```excel
=FILTRE(A:D;B:B="Inexistant")
```

Retourne #CALC! si aucun resultat.

### Solution

```excel
=FILTRE(A:D;B:B="Inexistant";"Aucun resultat")
```

## Erreur 4 : Tailles de plages differentes

### Probleme

```excel
=FILTRE(A1:A100;B1:B50="Oui")
```

Les plages n'ont pas la meme taille.

### Solution

Assurez-vous que les plages ont le meme nombre de lignes.

## Erreur 5 : Imbriquer sans precaution

### Probleme

```excel
=TRIER(FILTRE(UNIQUE(...)))
```

Difficile a debuguer si erreur.

### Solution

Utilisez LET pour decomposer :

```excel
=LET(
  unique;UNIQUE(A:A);
  filtre;FILTRE(unique;unique<>"");
  TRIER(filtre)
)
```

## Erreur 6 : Reference circulaire indirecte

### Probleme

Une formule dynamique reference sa propre zone de propagation.

### Solution

Placez la formule dans une zone isolee.

## Erreur 7 : Confondre TRIER et TRIERPAR

### TRIER

Trie sur une colonne du tableau lui-meme.

### TRIERPAR

Trie selon un tableau externe.

```excel
=TRIERPAR(Noms;Ages)    ' Trie Noms selon Ages
```

## Erreur 8 : Index incorrect dans PRENDRE

### Probleme

```excel
=PRENDRE(A1:D10;5;3)
```

L'index depasse la taille du tableau.

### Solution

Verifiez les dimensions avec LIGNES et COLONNES.

## Erreur 9 : Attendre un resultat unique

### Probleme

```excel
=RECHERCHEX(A1;B:B;C:D)
```

Retourne plusieurs colonnes alors qu'on n'en attendait qu'une.

### Solution

Specifiez une seule colonne de retour ou utilisez INDEX pour extraire.

## Erreur 10 : Ignorer l'ordre des arguments

### FILTRE

```excel
=FILTRE(donnees;condition;si_vide)
```

### UNIQUE

```excel
=UNIQUE(tableau;par_colonne;exactement_une_fois)
```

Les arguments optionnels ont un sens precis.

## Tableau recapitulatif

| Erreur | Prevention |
|--------|------------|
| #PROPAGATION | Liberer l'espace |
| #CALC! | Ajouter si_vide |
| Performance | Limiter les plages |
| Confusion | Utiliser LET |

## Bonnes pratiques

1. **Utilisez** LET pour decomposer
2. **Prevoyez** l'espace necessaire
3. **Testez** avec peu de donnees
4. **Documentez** les formules complexes
