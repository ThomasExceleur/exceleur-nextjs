# Debuter avec Power Pivot

Guide pour creer votre premier modele de donnees Power Pivot.

## Qu'est-ce que Power Pivot ?

Power Pivot permet de :
- Analyser des millions de lignes
- Relier plusieurs tables
- Creer des calculs avances (DAX)
- Alimenter des TCD puissants

## Activer Power Pivot

### Etapes

1. **Fichier** > **Options** > **Complements**
2. En bas : **Gerer** > **Complements COM**
3. **Atteindre**
4. Cochez **Microsoft Power Pivot for Excel**
5. **OK**

## Votre premier modele

### Etape 1 : Preparer les donnees

Creez deux tables :

**Table Ventes**
| ID | Date | ProduitID | Montant |
|----|------|-----------|---------|
| 1 | 01/01 | P001 | 100 |
| 2 | 02/01 | P002 | 200 |

**Table Produits**
| ProduitID | Nom | Categorie |
|-----------|-----|-----------|
| P001 | Widget | A |
| P002 | Gadget | B |

### Etape 2 : Ajouter au modele

1. Selectionnez chaque table
2. **Power Pivot** > **Ajouter au modele de donnees**

### Etape 3 : Creer la relation

1. **Power Pivot** > **Gerer**
2. Vue **Diagramme**
3. Faites glisser ProduitID de Ventes vers Produits

## Creer un TCD avec le modele

### Etapes

1. **Insertion** > **Tableau croise dynamique**
2. Cochez **Utiliser le modele de donnees de ce classeur**
3. **OK**
4. Choisissez des champs des deux tables

## Premiere mesure DAX

### Dans Power Pivot

1. Cliquez dans une cellule de la zone de calcul
2. Tapez :

```dax
Total Ventes = SUM(Ventes[Montant])
```

3. Appuyez sur Entree

### Utiliser la mesure

Elle apparait dans les champs du TCD.

## Types de relations

### Un-a-plusieurs

La plus courante. Un produit peut avoir plusieurs ventes.

### Plusieurs-a-plusieurs

Via une table intermediaire.

## Colonnes calculees vs Mesures

| Element | Quand utiliser |
|---------|----------------|
| Colonne calculee | Valeur pour chaque ligne |
| Mesure | Agregation (somme, moyenne...) |

## Formules DAX essentielles

### Somme

```dax
=SUM(Table[Colonne])
```

### Moyenne

```dax
=AVERAGE(Table[Colonne])
```

### Compte

```dax
=COUNTROWS(Table)
```

### Condition

```dax
=IF(condition; valeur_si_vrai; valeur_si_faux)
```

## Bonnes pratiques

1. **Nommez** clairement vos tables
2. **Creez** une table Dates
3. **Preferez** les mesures aux colonnes
4. **Documentez** vos formules

## Erreurs courantes

### Relation manquante

Si les donnees ne correspondent pas, verifiez les relations.

### Types incompatibles

Les colonnes liees doivent avoir le meme type.
