# Erreur #REF! dans Excel

L'erreur #REF! indique une reference de cellule invalide.

## Causes principales

### 1. Suppression de cellules referencees

Vous avez supprime une ligne ou colonne utilisee dans une formule.

### 2. Copie de formule hors limites

La formule copiee reference des cellules qui n'existent pas.

### 3. RECHERCHEV avec index trop grand

```excel
=RECHERCHEV(A1;B:C;5;FAUX)    ' Erreur : index 5 mais seulement 2 colonnes
```

### 4. Lien vers un classeur ferme invalide

Le fichier source a ete deplace ou supprime.

## Solutions

### Solution 1 : Annuler immediatement

**Ctrl + Z** pour annuler la suppression.

### Solution 2 : Corriger manuellement

1. Identifiez les cellules avec #REF!
2. Editez chaque formule (F2)
3. Remplacez #REF! par la bonne reference

### Solution 3 : Rechercher et remplacer

1. **Ctrl + H**
2. Rechercher : `#REF!`
3. Remplacer par : la reference correcte

## Prevention

### Utiliser des noms definis

```excel
=SOMME(Ventes)
```

Au lieu de :
```excel
=SOMME($A$1:$A$100)
```

### Verifier avant suppression

1. Selectionnez les cellules a supprimer
2. **Formules** > **Reperer les dependants**
3. Si des fleches apparaissent, attention aux formules

### Utiliser INDIRECT

```excel
=INDIRECT("A"&B1)
```

Moins sensible aux deplacements (mais attention aux performances).

## Cas specifique : RECHERCHEV

### Erreur courante

```excel
=RECHERCHEV(A1;B:D;5;FAUX)
```

L'index 5 depasse le nombre de colonnes (3).

### Correction

```excel
=RECHERCHEV(A1;B:D;3;FAUX)
```

## Localiser les erreurs #REF!

### Methode

1. **Ctrl + G** (Atteindre)
2. **Special...**
3. Cochez **Formules** > **Erreurs**
4. **OK**

## Bonnes pratiques

1. **Sauvegardez** avant de supprimer
2. **Utilisez** des noms definis
3. **Verifiez** les dependances
4. **Annulez** immediatement si erreur
