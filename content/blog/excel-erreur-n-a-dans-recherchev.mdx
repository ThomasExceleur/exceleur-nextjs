# Erreur #N/A dans RECHERCHEV : Solutions

L'erreur #N/A dans RECHERCHEV indique que la valeur recherchee n'a pas ete trouvee. Voici les causes et solutions.

## Signification de #N/A

#N/A signifie "Not Available" (Non Disponible). La valeur cherchee n'existe pas dans la plage de recherche.

## Cause 1 : Valeur inexistante

### Symptome

La valeur n'existe tout simplement pas dans les donnees.

### Solution

Verifiez que la valeur existe bien dans la premiere colonne de votre table.

### Masquer l'erreur

```excel
=SIERREUR(RECHERCHEV(A1; B:D; 2; FAUX); "Non trouve")
```

## Cause 2 : Espaces invisibles

### Symptome

La valeur semble identique mais n'est pas trouvee.

### Detection

Comparez les longueurs :
```excel
=NBCAR(A1)
=NBCAR(B1)
```

### Solution

```excel
=RECHERCHEV(SUPPRESPACE(A1); B:D; 2; FAUX)
```

Ou nettoyez les donnees sources.

## Cause 3 : Majuscules/Minuscules

### Note

RECHERCHEV n'est PAS sensible a la casse par defaut.

"Paris" = "PARIS" = "paris"

### Si vous avez besoin de sensibilite

Utilisez INDEX + EQUIV avec EXACT :
```excel
=INDEX(C:C; EQUIV(VRAI; EXACT(A1; B:B); 0))
```

## Cause 4 : Texte vs Nombre

### Symptome

Un nombre en texte ne correspond pas au meme nombre.

### Detection

```excel
=TYPE(A1)  ' 1 = nombre, 2 = texte
```

### Solutions

#### Convertir en nombre

```excel
=RECHERCHEV(CNUM(A1); B:D; 2; FAUX)
```

#### Convertir en texte

```excel
=RECHERCHEV(TEXTE(A1; "0"); B:D; 2; FAUX)
```

## Cause 5 : Format de date different

### Symptome

Les dates ne correspondent pas.

### Solution

Assurez-vous que les deux dates sont au meme format :
```excel
=RECHERCHEV(DATEVAL(A1); B:D; 2; FAUX)
```

## Cause 6 : Mauvaise plage

### Symptome

La colonne de recherche n'est pas la premiere de la table.

### Regle RECHERCHEV

La valeur cherchee DOIT etre dans la PREMIERE colonne de table_matrice.

### Solution

Reorganisez votre table ou utilisez INDEX + EQUIV.

## Cause 7 : Argument FAUX manquant

### Symptome

Resultat incorrect au lieu de #N/A.

### Cause

RECHERCHEV en mode approximatif (VRAI ou omis) avec donnees non triees.

### Solution

Ajoutez FAUX pour une correspondance exacte :
```excel
=RECHERCHEV(A1; B:D; 2; FAUX)
```

## Masquer l'erreur

### SIERREUR

```excel
=SIERREUR(RECHERCHEV(...); "Non trouve")
=SIERREUR(RECHERCHEV(...); 0)
=SIERREUR(RECHERCHEV(...); "")
```

### SI.NON.DISP (specifique #N/A)

```excel
=SI.NON.DISP(RECHERCHEV(...); "Alternative")
```

Preferable car les autres erreurs restent visibles.

## Utiliser RECHERCHEX (Excel 365)

### Avantages

- Valeur par defaut integree
- Plus flexible

### Syntaxe

```excel
=RECHERCHEX(A1; B:B; C:C; "Non trouve")
```

Le 4e argument est la valeur si non trouve.

## Verifier avant la recherche

### Avec NB.SI

```excel
=SI(NB.SI(B:B; A1)>0; RECHERCHEV(A1; B:D; 2; FAUX); "Inexistant")
```

## Debugging

### Etape par etape

1. Verifiez que la valeur existe
2. Comparez les longueurs (espaces)
3. Verifiez les types (texte vs nombre)
4. Verifiez la plage (premiere colonne)
5. Verifiez l'argument FAUX

### Formule de test

```excel
=EQUIV(A1; B:B; 0)
```

Retourne la position ou #N/A.

## Resume des solutions

| Cause | Solution |
|-------|----------|
| Valeur inexistante | SIERREUR |
| Espaces | SUPPRESPACE |
| Texte/Nombre | CNUM ou TEXTE |
| Mauvaise plage | Corriger la plage |
| Mode approximatif | Ajouter FAUX |

## Bonnes pratiques

1. **Toujours** utiliser FAUX pour correspondance exacte
2. **Nettoyer** les donnees (espaces, formats)
3. **Gerer** les erreurs avec SIERREUR
4. **Preferer** RECHERCHEX si disponible
