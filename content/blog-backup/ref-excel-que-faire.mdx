# Erreur #REF! dans Excel

L'erreur #REF! indique une référence de cellule invalide.

## Causes principales

### 1. Suppression de cellules referencees

Vous avez supprime une ligne où colonne utilisée dans une formule.

### 2. Copie de formule hors limites

La formule copiee référence des cellules qui n'existent pas.

### 3. RECHERCHEV avec index trop grand

```excel
=RECHERCHEV(A1;B:C;5;FAUX)    ' Erreur : index 5 mais seulement 2 colonnes
```

### 4. Lien vers un classeur ferme invalide

Le fichier source à ete déplacé où supprime.

## Solutions

### Solution 1 : Annuler immédiatement

**Ctrl + Z** pour annuler la suppression.

### Solution 2 : Corriger manuellement

1. Identifiez les cellules avec #REF!
2. Éditez chaque formule (F2)
3. Remplacez #REF! par la bonne référence

### Solution 3 : Rechercher et remplacer

1. **Ctrl + H**
2. Rechercher : `#REF!`
3. Remplacer par : la référence correcte

## Prevention

### Utiliser des noms définis

```excel
=SOMME(Ventes)
```

Au lieu de :
```excel
=SOMME($A$1:$A$100)
```

### Vérifier avant suppression

1. Sélectionnez les cellules à supprimer
2. **Formules** > **Reperer les dependants**
3. Si des fleches apparaissent, attention aux formules

### Utiliser INDIRECT

```excel
=INDIRECT("A"&B1)
```

Moins sensible aux deplacements (mais attention aux performances).

## Cas spécifique : RECHERCHEV

### Erreur courante

```excel
=RECHERCHEV(A1;B:D;5;FAUX)
```

L'index 5 dépasse le nombre de colonnes (3).

### Correction

```excel
=RECHERCHEV(A1;B:D;3;FAUX)
```

## Localiser les erreurs #REF!

### Méthode

1. **Ctrl + G** (Atteindre)
2. **Special...**
3. Cochez **Formules** > **Erreurs**
4. **OK**

## Bonnes pratiques

1. **Sauvegardez** avant de supprimer
2. **Utilisez** des noms définis
3. **Vérifiez** les dependances
4. **Annulez** immédiatement si erreur
